# Research Pipeline (Ресерч-тян)

Исследовательский пайплайн для глубокого анализа любых тем с верификацией и структурированным выводом.

## Триггерные фразы

Пайплайн активируется когда пользователь говорит:
- "загрузи мой ресерч пайплайн"
- "запусти ресерч-тян"
- "исследуй тему X"
- "найди информацию про X"
- "ресерч пайплайн"
- "проведи исследование по X"
- "изучи тему X"

## Способы запуска

### Вариант 1: Через orchestration (рекомендуется)

```
/orchestration:template research-pipeline
```

Затем ввести тему для исследования когда будет запрошен параметр `topic`.

### Вариант 2: Прямой запуск workflow

```
/orchestration:run orchestration:research-communicator:"получить запрос" -> orchestration:research-senior:"исследовать тему X" -> orchestration:research-editor:"проверить и оформить"
```

### Вариант 3: Через Task tool (legacy)

Последовательно запустить агентов:

1. **research-senior** (opus) — Глубокий сбор информации
2. **research-editor** (sonnet) — Верификация и дополнение
3. **research-communicator** (sonnet) — Передача результатов

## Агенты пайплайна

| Агент | Модель | Роль |
|-------|--------|------|
| research-senior | opus | Классификация, уточнение, глубокий поиск через Tavily/Reddit |
| research-editor | sonnet | Проверка на актуальность, противоречия, полноту |
| research-communicator | sonnet | Общение с пользователем, финальный вывод |

## Порядок выполнения

```
research-communicator (уточнение запроса)
    ↓
research-senior (классификация + вопросы)
    ↓
research-senior (глубокое исследование → черновик)
    ↓
research-editor (верификация + чек-лист)
    ↓
@review (одобрение пользователя)
    ↓
research-editor (итоговый документ)
    ↓
research-communicator (уведомление о завершении)
```

## Метки в выводе

- ✅ — Подтверждённая информация
- ⚠️ — Требует дополнительной проверки
- ❌ — Не найдено / противоречиво

## Восстановление после обновления плагина

Если агенты перестали работать после обновления orchestration:

```bash
~/.claude/scripts/restore-orchestration-agents.sh
```

## Пример использования

Пользователь: "Исследуй тему настройки MCP серверов в Claude Code"

Claude:
1. Запускает `/orchestration:template research-pipeline` с параметром `topic="настройка MCP серверов в Claude Code"`
2. Агенты последовательно выполняют исследование
3. Результат выводится пользователю
