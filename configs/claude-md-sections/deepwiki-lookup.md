
## DeepWiki Lookup

Автоматический разбор репозиториев через DeepWiki.

### Когда активируется
Когда пользователь даёт ссылку на GitHub-репозиторий (например `https://github.com/owner/repo`, `github.com/owner/repo`, или просто `owner/repo` в контексте обсуждения репо):

### Алгоритм
1. Извлечь `owner/repo` из ссылки
2. Запросить `https://deepwiki.com/{owner}/{repo}` через `tavily_extract` (extract_depth: advanced)
3. Если страница найдена — дать структурированный разбор:
   - **Что это** — краткое описание проекта
   - **Архитектура** — ключевые компоненты и их связи
   - **Стек** — языки, фреймворки, зависимости
   - **Ключевые модули** — основные части кодовой базы
   - **Как использовать** — быстрый старт (если есть)
4. Если страница не найдена или пустая — сообщить, что репо не проиндексирован на DeepWiki
5. Для глубокого анализа — дополнительно запросить подстраницы (архитектура, API и т.д.)

### Примечания
- Работает только для публичных репозиториев
- Не все репо проиндексированы — популярные есть почти всегда
- Данные могут отставать от текущего состояния репо
