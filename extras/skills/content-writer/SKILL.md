# Content Writer Skill

## Описание
Создание постов и контента на основе идей пользователя. Использует веб-поиск для актуальной информации и анализирует референсы для сохранения авторского стиля и структуры.

## Когда использовать
Триггерные фразы:
- "написать текст"
- "создать текст"
- "мне нужна помощь в написании поста"
- "я хочу создать пост о [тема]"
- "давай напишем пост"
- "накидать тему"
- "нужен черновой вариант"

## Настройка скилла

### Референсы для стиля
Создайте папку `references/` внутри папки этого скилла и поместите туда примеры текстов в нужном стиле:
- `references/` — примеры постов, статей, гайдов
- Файлы должны быть в формате `.md`
- Назовите файлы описательно: `01-гайд-по-настройке.md`, `02-обзор-продукта.md`

Скилл будет анализировать эти файлы для понимания:
- Структуры текста
- Стиля изложения
- Тональности
- Формата контента

### Папка результатов
Скилл автоматически создаст папку `AIText/` для сохранения готовых постов.

## Рабочий процесс

### 1. Получение темы
Принимает тему в любом формате:
- "Хочу написать обзор нового AI-инструмента"
- "Думаю показать, как пользоваться платформой в 2026"
- "Нужен пост про актуальные тарифные планы"

### 2. Исследование
1. **Веб-поиск** — собирает актуальную информацию по теме
2. **Анализ референсов** — изучает примеры из `references/` для стиля и структуры
3. **Определение формата** — гайд, размышления, обзор (на основе темы)

### 3. Создание черновика
**Стандартная структура:**
- **Провокационный заголовок** — цепляющий, привлекающий внимание
- **Хук** — сразу к делу, без воды
- **Нарастание аргументов** — логическое развитие с примерами
- **Заключение + призыв к действию** — яркое завершение

**Адаптация стиля:**
- Анализирует референсы из `references/` для определения тона
- Подстраивается под формат (гайд, обзор, размышления)
- Сохраняет консистентность с примерами

**Объём:**
- **По умолчанию:** близко к лимиту Telegram (~4000 символов)
- **Адаптируется под запрос** — может делать короче или длиннее
- **Подстраивается под тему** — сложные темы длиннее, простые короче

### 4. Итерации и правки
После отправки черновика:
- Ждёт обратной связи от пользователя
- Отвечает на правки — перерабатывает указанные части
- Задаёт уточняющие вопросы если что-то неясно
- Показывает обновлённый вариант

### 5. Завершение и сохранение
Когда пользователь доволен результатом, сохраняет финальную версию:
```
AIText/YYYY-MM-DD-тема-поста.md
```

**Формат файла:**
```markdown
# Заголовок поста

*Дата создания: DD.MM.YYYY*
*Тема: [описание темы]*

[Итоговый текст поста]
```

## Технические детали

### Необходимые инструменты
- `web_search` — для актуальной информации
- `read` — для анализа референсов
- `write` — для сохранения итогового поста

### Структура файлов
```
content-writer/
├── SKILL.md (этот файл)
├── references/
│   ├── 01-пример-гайда.md
│   ├── 02-пример-обзора.md
│   └── 03-пример-размышлений.md
└── (скилл создаст автоматически)
    └── AIText/
        ├── 2026-01-30-название-поста.md
        └── 2026-01-31-другой-пост.md
```

## Правила форматирования

### Рекомендуемый формат:
- **Заголовки** — только жирный текст **Заголовок**
- **Списки** — короткие черточки: •
- **Объяснения** — любой технический термин объяснять простыми словами
- **Конкретика** — давать конкретные советы, а не общие слова
- **Логичные призывы к действию** — CTA должен вытекать из содержания

### Структура поста:
```
**Провокационный заголовок**

Хук-абзац...

**Подзаголовок раздела**
• Пункт списка
• Пункт списка

Основной текст...

Призыв к действию
```

## Кастомизация

Вы можете адаптировать этот скилл под свои нужды:
1. **Добавьте свои референсы** в папку `references/`
2. **Измените триггерные фразы** в разделе "Когда использовать"
3. **Настройте правила форматирования** под свою платформу
4. **Адаптируйте структуру** под свой стиль

## Важные замечания
- Скилл адаптируется под стиль референсов из папки `references/`
- Всегда ищет актуальную информацию через веб-поиск
- Работает итеративно до полного удовлетворения результатом
- Сохраняет только итоговые версии, черновики не архивирует
- Автоматически создаёт необходимые папки если их нет
